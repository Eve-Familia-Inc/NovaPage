Nova｜Python3.9｜多分モダンなWebアプリケーションフレームワーク

Novaは、よくある普通のWebアプリケーションのためにPython3.9で書かれた普通のフレームワークです。
今ではすっかりスタンダードとなった非同期処理(Async)を基本とし、Html, CSS, 画像などの静的コンテンツをオンメモリ化。　
WebSocketも気軽に利用できて、分かりやすく自由度の高いコードが書けます。 マッピングのために妙なデコレーターを利用することはせず、サーバー部分で最後にメソッドを寄せ集めます。 特に何も考えなくても機能単位でコードの分割がしやすく、チームでの開発もスムーズに進むでしょう。

インストール！

Pythonのバージョンは3.7以上である必要があります。
環境によってはpipがPython2系で動作する可能性があります。その場合はpip3を利用しましょう。 

pip install Nova-EveFamilia

ネコでもわかる簡単なサンプル

import Nova


# エコーサンプル POSTパラメーター用
async def PostTest(self, connection, Request, ReplyHeader):
    # レスポンス用の構造体を編集しましょう。正しいレスポンスを返すために、これら3つの設定は必須です。
    ReplyHeader["ReplyContent"] = b"POST:" + Request["content"]
    ReplyHeader["Content-Type"] = b"text/html"
    ReplyHeader["Status"] = 200
    # レスポンスを返します
    await self.Reply(connection, ReplyHeader)


# エコーサンプル GETパラメーター用
async def GetTest(self, connection, Request, ReplyHeader):
    # レスポンス用の構造体を編集しましょう。正しいレスポンスを返すために、これら3つの設定は必須です。
    ReplyHeader["ReplyContent"] = b"GET:" + Request["content"]
    ReplyHeader["Content-Type"] = b"text/html"
    ReplyHeader["Status"] = 200
    # レスポンスを返します
    await self.Reply(connection, ReplyHeader)


# IP:127.0.0.1 PORT:7777
# /var/www/html 以下のファイルからMIMEタイプに一致するものだけ自動的にメモリに格納します
nova = Nova.Service(
    host="127.0.0.1",
    port=7777,
    root_path="/var/www/html"
)

# パスにメソッドをマッピング
nova.PostFunctions = {
    "/PostTest.post": PostTest
}
nova.GetFunctions = {
    "/GetTest.get": GetTest
}

# SSLを有効化
nova.EnableSSL(
    domain_cert="domain.cert.pem",
    private_key="private.key.pem"
)

# あとはスタートするだけ！
nova.Start()

特徴

非同期通信

昔はselect()を使ってノンブロッキングソケットをぐるぐる回していたものですが、近頃は便利になったものです。 非同期通信によって、送受信が完了していなくとも別の処理を実行することが出来ます。古き良きC10K問題もこれで解決です。

静的コンテンツのオンメモリ化

最近のハードウェアはびっくりするほど超高速です。体感的な速度はあまり変わらず、 ディスクアクセス速度に制約のあるシステムでは有用かもしれません。

WebSocket

WebSocketというのは面白い技術です。 少々クセがあるからかあまり盛り上がりを見せてはいませんが、触れてみると案外新しい発見があるかもしれません。

高い自由度

Novaは一般的なオブジェクト思考プログラムと違い、高度な抽象化を提供しません。だからこそコンテンツや機能を動的に追加したい、そんな痒いところに手がなかなか届きそうに無い時でも、もうわざわざ検索する必要はありません。 VirtualHostだって思いのままに実現できます。

それなりの拡張性

デフォルトのNovaには大した機能はありません。しかしNovaそのものはオブジェクトのかたまりでコードもそんなに長くないため、Pythonの自由さも相まって機能の追加は容易です。 今後アドオンの開発も簡単にできるよう整備を進めていきます。

快適なチーム開発

他のフレームワークがどうかというのは正直あまり知らないのですが、Novaなら役割ごとのコード分離が直感的に行えるため、設計や実装でストレスを感じることが無くなるかもしれません。

設計思想

データを追い、コードの先を歩く

大切なのはコードの先にあるデータであり、それが最も重要な本質だと考えます。
